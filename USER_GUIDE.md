## Что это такое

**cvr.name.coder** — это AI-ассистент для разработки, который работает прямо внутри VS Code как боковая панель (sidebar extension). Основные возможности:

- **Потоковые ответы** — текст появляется в реальном времени, токен за токеном
- **Мульти-провайдерность** — Gemini, OpenAI, Anthropic, DeepSeek, Groq, OpenRouter, Mistral, BaseTen, локальные модели через Ollama
- **Автономный агент** — может самостоятельно писать код, запускать команды, коммитить в Git, искать по проекту
- **Goal Loop** — задаёте цель (`/goal`), агент работает автономно, пока не достигнет результата или не исчерпает бюджет итераций
- **Постоянная память** — проектная MEMORY.md и пользовательские настройки USER.md
- **Undo / Redo** — отмена любых изменений, внесённых агентом
- **MCP инструменты** — подключение внешних серверов инструментов
- **Project Oracle RAG** — авто-индексация проекта для семантического поиска и retrieval
- **Code Review** — обзор git diff с категоризированными замечаниями AI
- **Browser automation** — Playwright-инструменты для навигации, скриншотов и проверки веба
- **Vision** — загрузка изображений в мультимодальные модели
- **Design systems** — работа с дизайн-системами и переиспользуемыми пресетами
- **Agent Marketplace** — публикация и переиспользование агентов, навыков, плагинов и правил
- **Team Sync** — синхронизация памяти и проектного контекста

Всё работает локально внутри VS Code, никаких внешних серверов не требуется.

## Установка

### Требования

- VS Code 1.85+
- Node.js 18+ (входит в состав VS Code)
- Git (рекомендуется)

### Установка из .vsix

1. Соберите или скачайте файл расширения:
   ```bash
   npm install
   npm run build
   npm run package:vscode
   # Результат: vscode/cvr-name-coder-1.x.x.vsix
   ```

2. Установите в VS Code:
   ```bash
   code --install-extension vscode/cvr-name-coder-1.x.x.vsix
   ```
   Или через интерфейс: `Ctrl+Shift+P` → **Extensions: Install from VSIX...** → выберите файл.

3. Перезагрузите VS Code: `Ctrl+Shift+P` → **Developer: Reload Window**

4. В Activity Bar (левая панель) появится иконка **cvr.name** — кликните по ней.

> **Обновление:** при установке новой версии `.vsix` старая заменяется автоматически, настройки сохраняются.

## Первый запуск и настройка

При первом открытии панели cvr.name вы увидите настройки. **Важно:** начиная с версии 1.5.0 дефолтный провайдер не выбран — нужно указать его явно.

### Настройка AI Engine

1. Нажмите на **шестерёнку (⚙️)** в правом верхнем углу панели.
2. Во вкладке **AI Engine** выберите:
   - **Provider** — облачный (Gemini, OpenAI, Anthropic...) или Local (Ollama)
   - **Model** — модель из списка или введите название вручную
   - **API Key** — ключ для выбранного провайдера
3. Нажмите **Save**.

> **Совет:** нажмите на иконку ключа (KeyRound) рядом с провайдером, чтобы открыть страницу получения API-ключа.

### Горячие клавиши

| Команда | Клавиши |
|---------|---------|
| Открыть cvr.name панель | `Ctrl+Shift+C` |
| Открыть полный Dashboard | `Ctrl+Shift+P` → `cvr.name: Launch Dashboard` |
| Inline редактирование | `Ctrl+K Ctrl+K` |

## Работа с чатом

### Базовое использование

1. Введите запрос в поле ввода.
2. Нажмите `Enter` или кнопку отправки.
3. Ответ появится потоково (токен за токеном).
4. Кнопка **Cancel** останавливает генерацию.

### Загрузка изображений

1. Включите **Vision** в настройках (Settings → Vision → ON).
2. Нажмите на иконку скрепки/изображения в поле ввода.
3. Выберите файл — поддерживается drag-and-drop.
4. Работает с Gemini, OpenAI GPT-4o, Anthropic Claude.

### Голосовой ввод

1. Включите в настройках: **Voice Input → ON**.
2. Выберите язык распознавания.
3. Нажмите на иконку микрофона в поле ввода.
4. Речь автоматически преобразуется в текст (опционально — автоотправка после паузы).

### Отмена изменений

Каждое действие агента (запись файла, правка, команда) фиксируется в истории изменений:

- **Undo** — отменить последнее действие агента (`Ctrl+Shift+Z` или кнопка в интерфейсе)
- **Redo** — повторить отменённое действие

## Слеш-команды

Введите в чате команду с `/` для специального режима:

| Команда | Описание | Пример |
|---------|----------|--------|
| `/analyze` | Глубокий анализ структуры проекта | `/analyze` |
| `/fix` | Найти и исправить проблемы в коде | `/fix src/utils.ts` |
| `/optimize` | Оптимизация сложности кода | `/optimize` |
| `/audit` | Аудит безопасности | `/audit` |
| `/explain` | Объяснить логику файла | `/explain src/App.tsx` |
| `/refactor` | Рефакторинг с сохранением функциональности | `/refactor` |
| `/review` | Code review текущего git diff | `/review` |
| `/undo` | Отменить последнее изменение | `/undo` |
| `/redo` | Повторить отменённое изменение | `/redo` |
| `/goal` | **Запустить автономную цель с judge-оценкой** | `/goal Реализовать авторизацию — JWT + bcrypt + middleware` |

### Goal команда (подробно)

Команда `/goal` запускает автономный цикл достижения цели:

```
/goal <описание цели> — <критерии успеха>
```

- **Цель** — что нужно сделать (обязательно)
- **Критерии успеха** — через `—` (длинное тире). Необязательно, но сильно повышает качество результата

**Примеры:**

```
/goal Добавить тёмную тему — Переключатель работает, CSS-переменные используются, предпочтение сохраняется в localStorage

/goal Написать тесты для useSettings — Покрытие >80%, все edge cases протестированы, моки корректны

/goal Рефакторинг API routes — Убран дублирующийся код, все тесты проходят, типы строгие
```

После отправки:
1. Открывается панель **Goal Loop** вместо чата
2. Агент работает автономно: думает → действует → judge оценивает прогресс
3. Панель показывает: статус, прогресс бар, итерации, шаги, вердикты judge
4. Вы можете **Abort** в любой момент
5. По завершении — результат и полная история шагов

**Ограничения цикла:**
- Максимум итераций: 50 (по умолчанию)
- Максимум токенов: 500 000
- Максимум времени: 2 часа
- Judge проверяет каждый шаг и даёт подсказку агенту

## Автономный Goal Loop

### Как это работает

```
Пользователь: /goal Сделать X
         ↓
    [GoalOrchestrator]
         ↓
    ┌─────────────┐
    │ 1. Think    │ ← AI думает, что делать
    └──────┬──────┘
           ↓
    ┌─────────────┐
    │ 2. Action   │ ← Выполняет инструмент (write_file, git commit...)
    └──────┬──────┘
           ↓
    ┌─────────────┐
    │ 3. Judge    │ ← Отдельный AI оценивает: достигнута ли цель?
    └──────┬──────┘
           ↓
    ┌─────────────┐
    │ 4. Hint     │ ← Если не достигнута — подсказка для следующего шага
    └──────┬──────┘
           ↓
      (повтор до COMPLETE или лимита)
```

### Статусы Goal Loop

| Статус | Значение |
|--------|----------|
| **RUNNING** | Агент работает |
| **COMPLETED** | Цель достигнута, judge подтвердил |
| **ERROR** | Исчерпан лимит итераций/токенов/времени или ошибка |
| **ABORTED** | Пользователь нажал Abort |

### Восстановление

При перезагрузке VS Code активные Goal-сессии автоматически восстанавливаются. Если goal был на паузе (paused) или в ошибке — можно возобновить через API или UI.

## Память и контекст

### MEMORY.md

Файл `.opencode-infinite/MEMORY.md` содержит знания агента о проекте:

- Архитектурные решения
- Ключевые факты о кодовой базе
- Цели и прогресс
- Правила, которые агент сам вывел

**Редактирование:**
- Через панель **Memory** в sidebar
- Агент сам дополняет MEMORY.md после каждых 5 сообщений (Dreamer Engine)

### USER.md

Файл `.opencode-infinite/USER.md` — ваши предпочтения:

```markdown
## Preferences
- Я предпочитаю функциональные компоненты
- TypeScript strict mode
- Tailwind CSS для стилей
- Vitest для тестов
```

Агент читает USER.md перед каждым ответом и адаптируется под ваш стиль.

### RAG (семантический поиск)

Автоматически индексируются все текстовые файлы проекта. Можно искать по смыслу:

```
Как в этом проекте настроен routing?
```

Агент найдёт релевантные файлы даже если в них нет слова "routing".

## Локальные модели (офлайн)

Для работы без интернета и API-ключей:

### Ollama

1. Установите [Ollama](https://ollama.com)
2. Скачайте модель:
   ```bash
   ollama pull llama3
   # или для кода:
   ollama pull codellama:7b
   # или:
   ollama pull qwen2.5-coder:14b
   ```
3. Запустите сервер:
   ```bash
   ollama serve
   ```
4. В настройках cvr.name:
   - Provider: **Local**
   - Local URL: `http://localhost:11434`
   - Model: `llama3` (или та, что вы скачали)

### LM Studio

1. Запустите LM Studio, включите сервер на порту `1234`
2. В настройках:
   - Provider: **Local**
   - Local URL: `http://localhost:1234`

### llama.cpp

1. Запустите сервер: `./server -m model.gguf --port 8080`
2. В настройках:
   - Provider: **Local**
   - Local URL: `http://localhost:8080`

> **Примечание:** локальные модели медленнее облачных, но полностью приватные.

## Разрешения и безопасность

### Система разрешений

Файл `.cvr/permissions.json` управляет тем, что агент может делать без спроса:

```json
{
  "rules": [
    { "pattern": "read_file", "action": "allow" },
    { "pattern": "write_file", "action": "ask" },
    { "pattern": "execute_command", "action": "ask" },
    { "pattern": "*.env*", "action": "deny" },
    { "pattern": "rm -rf", "action": "deny" }
  ],
  "defaultAction": "ask"
}
```

| Действие | Значение |
|----------|----------|
| **allow** | Выполнять без подтверждения |
| **ask** | Показывать диалог подтверждения |
| **deny** | Запретить полностью |

### Где хранятся ключи

- **В режиме dev-сервера:** в файле `.env` (не коммитится, указан в `.gitignore`)
- **В VS Code расширении:** в `localStorage` внутри VS Code (локально на вашем компьютере)
- Ключи никуда не отправляются, кроме API провайдера

### Безопасность файлов

- Агент не может выйти за пределы рабочей директории (path traversal защита)
- Запрещены операции с системными файлами
- `.env` файлы по умолчанию защищены (deny)

## Поиск и устранение неисправностей

### AI не отвечает / пустой экран

| Симптом | Причина | Решение |
|---------|---------|---------|
| "AI provider not configured" | Не выбран провайдер | Откройте Settings (⚙️), выберите Provider и Model, введите API-ключ |
| "API key invalid" | Неверный ключ | Проверьте ключ, обновите в настройках |
| "Model not found" | Ошибка в названии модели | Выберите из списка или проверьте правильность написания |
| Пустой ответ | Ошибка сети / провайдер недоступен | Перезагрузите VS Code (`Ctrl+Shift+P` → Reload Window) |

### Локальная модель не найдена

1. Проверьте, что Ollama/LM Studio запущен:
   ```bash
   ollama serve
   # или
   curl http://localhost:11434/api/tags
   ```
2. Проверьте URL в настройках — должен совпадать с портом сервера
3. Убедитесь, что модель скачана: `ollama list`

### Goal Loop завис

1. Нажмите кнопку **Abort** в панели Goal
2. Или перезагрузите VS Code — состояние сохранится, можно будет возобновить

### Не работает Undo

1. История изменений хранится в `.opencode-infinite/changes.json`
2. Если файл повреждён — удалите его (история сбросится)
3. Проверьте права на запись в `.opencode-infinite/`

### Медленные ответы

- **Gemini 2.5 Flash** — самый быстрый и дешёвый
- **Local модели** — медленнее, особенно на CPU
- Уменьшите **maxTokens** в настройках
- Включите **Response caching** для повторяющихся запросов

### Ошибки типов / сборки

Если агент написал код с ошибками TypeScript:
1. Попросите исправить: `/fix файл.ts`
2. Или запустите `npm run type-check` и скопируйте ошибки в чат

## Быстрые советы

1. **Начинайте с конкретики** — вместо "исправь баги" напишите "исправь ошибку типа в src/hooks/useAuth.ts, где user может быть null"
2. **Используйте `/goal`** для задач, которые требуют >3 шагов — агент сам справится лучше, чем пошаговый чат
3. **Пишите критерии успеха** — чем конкретнее `—` часть в `/goal`, тем точнее результат
4. **Проверяйте MEMORY.md** — агент помнит архитектуру, но иногда устаревает; обновите вручную через панель Memory
5. **Git коммиты** — агент может коммитить автоматически; настройте auto-commit в Settings, если доверяете
6. **Skills** — если агент часто делает одно и то же (например, создаёт React компоненты), он автоматически создаст skill; используйте их через панель Skills
